# CourseFlow Documents

This folder is the handoff package for a new teammate. It is separate from the repo's existing `docs/` folder, which mostly stores import datasets and templates.

## Start Here

If you are brand new to CourseFlow, read these in order:

1. `developer-onboarding.md`
2. `architecture-overview.md`
3. `architecture-visuals.md`
4. `operations-and-data.md`
5. `product-user-guide.md`

## What Each Document Covers

- `developer-onboarding.md`
  - local setup
  - how to run the app
  - key commands
  - first-day checklist
- `architecture-overview.md`
  - frontend/backend structure
  - major modules
  - how data moves through the system
- `architecture-visuals.md`
  - system diagrams
  - request flow visuals
  - startup/runtime diagrams
  - data import and seeding visuals
  - product navigation map
- `operations-and-data.md`
  - database modes
  - environment variables
  - seeding and import data
  - common reset/recovery workflows
- `product-user-guide.md`
  - what the product does
  - how users move through the major features
  - what each page is for

## Intended Outcome

After reading this folder, a new person should be able to:

- run CourseFlow locally
- understand the major app areas
- know where code lives
- know how data gets into the system
- debug the most common environment issues
- contribute safely without needing a live walkthrough first
